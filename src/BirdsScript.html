つい、カッ！となってLL言語を作りはじめてしまいました。名前は「BirdsScript」です。
（2013/08/10 現在、未完成。）

特徴：

すべてのプログラマーのために「軽量で」「安全で」「大規模開発に耐えられる」楽しい言語を目指して設計してみました。
「軽量で」「安全で」「大規模開発に耐えられる」理由は、

　・定数宣言を重視（不必要な変数はバグの元となると考えて）

　・継承は存在しない（実装の継承は過度に隠ぺいされて可読性が悪いと考えて）

　・ポインターは存在しない（ポインターは、バグの温床という考え）

　・GCではなく、スコープで解放され、デストラクタが走る。

　・型名はほとんんど記述する必要がない。

　・コンテナが存在する

　・関数とクラスは同じ記述、同じ実装。

　・無名関数がある。

　・実装が軽量。

　・タイプセーフな言語にしたい思いは強いです。

　・テストも重視。デバッグも重視。とにかく、プログラマが苦労しない言語を目指しています。


概要：

コンパイラというか中間言語へのトランスレータ「Yamasemi」と、
中間言語の実行部(いわゆるVM)「Kawasemi」の２段構成になっています。

現状、言語設計+試験期間な意味からコンパイラよりもインタープリタに近い状態です。現在は実装の確認という意味で JavaScript 書いています。
今後の予定としては、やはり C/C++ への移植です。JavaScriptで 1000行程度なので、C/C++ でも 4000行程度の軽量な実装になると予想しています。

型は現在、整数型(int)と文字列型(str)しかありません。早めに、bool, enum はサポートしたいと思っています。
float やその他の型は外部のライブラリとして用意するつもり。（言語そのもののの実装を軽くしたいので。）

クラスやオブジェクトのようなものは JavaScript と同じようなプロトタイプで用意する方向です。
継承は嫌いなので、おそらく永遠にサポートされないでしょう。

逆に STL のようなジェネリックプログラミングは優先度高めでサポートします。
コンテナというか「リスト（配列）」や「マップ（連想配列）」は優先度を上げて実装しています。現状は中途半端に実装されています。

そしてテストとデバッグのしやすさを売りにしたいので test() や assert(), trace() とかを早めに実装したい。

以下、「BirdsScript」の文法です
-----------------------------------------------------------------
■コメント

    '---' いわゆる C++ の // と同じ
    '...' 上と同じ

■ 定数/変数宣言

// 予約語は ':', '::'
// 必ず初期値が必用。空の変数や定数は存在しない（させない）ことで、バグが入り込まないようにする設計方針。
// 定数 いわゆるC++の const。Scalaの val とも言う。値を変えることができません。

    i : 0 ; ... 定数宣言 val

    str_n : "string\n" ; ... エスケープ有りの文字列

    str : 'string' ; ... エスケープ無しの文字列

// ':' は const なので定数の宣言、上書きするには変数宣言 '::'。 いわゆる var。
// 変数の方が制約が緩い（危険）なので、ハイコストにしています。上手に使ってください。

    i :: 0 ; ... 変数宣言 var

    str :: "string" ;

■ if 文

// 'if', 'else' という予約語はありません。C言語の三項演算子のような '??', '!!' を使います。elseは':'ではなく '!!' な点に注意してください。

    foo < 0 ?? foo = 0 ;

    foo < 0 ?? foo = 0 !! foo = 1 ;

    boo : ( foo < 0 ?? 0 !! 1 ) ; ... スコープとの組み合わせ


// ?? の前は必ず比較文であること。比較文以外はエラーになります。

    is_ok ?? a=0 !! a=1 ; ... NG

    is_ok == true ?? a=0 !! a=1 ; ... OK



// 現在のところ else 文のみの書き方はNGになっています。メリットが感じられるようなら実装を検討します。

    foo >= 0 !! foo = 0 ; ... NG

■ スコープ（ブロックスコープ）

// いわゆるC言語の '{','}' ではなくて '(', ')' になります。
// 変数/定数宣言のスコープが制限できるので上手に使ってください。

    foo < 0
    ?? ( foo = 0 ; boo = 0 ; woo = 0 ; )
    !! ( foo = 1 ; boo = 1 ; woo = 1 ; )

    foo < 0
    ?? (
        foo = 0 ;
        boo = 0 ;
        woo = 0 ;
    )
    !! (
        foo = 1 ;
        boo = 1 ;
        woo = 1 ;
    )


// スコープ'(',')'そのものに値を持たせることもできます。
// 予約語は'out'。return文に似ているけど、処理は中断されないので注意。
// ちょっと他の言語には無い考えなので、ここではスキップしておっけーです。

    a : (
        foo = 1 ;
        out : boo = 2 ;  ... この式が値が()の値になる
        woo = 3 ;
    ) ;
    @.co << a ;          ... a : 2 になる。 @.co は C言語の stdout の意味で仮実装中。

    b : (
        foo = 1 ;
        boo = 2 ;
        woo = 3 ;
        boo            ... outを書かなくても最後の';'を書かなければオッケー。
    ) ;
    @.co << b ;        ... b : 2 になる。

■ for 文

// '{{', '}}' はループ文 -> for(;;){ ... }。
// break は値を返すことができる。無しも可。
// ちなみに無限ループではなくて、リミット上限に到達すると例外が発生してループを終わります。
// break文が無いとワーニングの予定（未実装）。できる限り安全側に倒したい。

    a : {{

        break 10 ;    ... ループを抜ける。 out = 10 ; break ; と同じ。

        continue ;    ... ここは実行されないけど。ループの先頭に戻る。

    }} ;
    @.co << a ; ... a : 10 ;



// '.{{', '}}' は each, 'it' はループ内で使用できる変数（イテレータの意味：予約語です）
// 変数のイテレータであれば、上書き可能です。

    5.{{ @.co << it << ', ' ; }}  ... 0, 1, 2, 3, 4

    @.co << 5.{{ out << it << ', ' ; }}  ... 0, 1, 2, 3, 4

    b : 10.{{
        it == 5 ?? break 555 ;
    }} ;
    @.co << b ; ... b = 555



    c : 'String' ;
    d : c.{{
        it >= 'a' && it <= 'z'
        ?? out << it - ( 'a' - 'A' ) ; ... out は変数なので上書き可能
    }} ;
    @.co << d ; ... d : 'STRING'



// for ( int i=0 ; i<10 ; i++ ) ... をあえて書くなら

    i :: 0 ;
    {{
        ...
        i >= 10 ?? break ;
        i ++ ;
    }}



// 以下、例外と同等な機能なので・・・実装検討中

// 多重ループを抜ける break, continue (構造化された goto)

    {{
        ...
        {{
            break a ; ... a のあるスコープの外へ飛ぶ
        }}

    [ a ]}}  ... <- ここへ飛ぶ（外へ抜ける）

    {{[ bb ] ... <- ここへ飛ぶ（ループを繰り返す）
        ...
        {{
            continue bb ; ... bb のあるスコープの最初に飛ぶ
        }}
    }}


// ループを抜ける break, continue (構造化された goto)

    (
        ...
        {{
            break a ; ... a のあるスコープの外へ飛ぶ
        }}

    [ a ])  ... <- ここへ飛ぶ（外へ抜ける）

    ([ bb ] ... <- ここへ飛ぶ（ループを繰り返す）
        ...
        {{
            continue bb ; ... bb のあるスコープの最初に飛ぶ
        }}

    )


// ループを抜ける break, continue (構造化された goto)

    (
        ...
        {{
            break a ; ... a のあるスコープの外へ飛ぶ
        }}

    [ a : { .. } ])   ... break 時に実行される処理が書ける  ... <- ここへ飛ぶ（指定された処理を実行して、外へ抜ける）

    ([ bb : { .. } ]  ... continue 時に実行される処理が書ける ... <- ここへ飛ぶ
        ...
        {{
            continue bb ; ... bb のあるスコープの最初に飛ぶ
        }}

    )

■ class と関数の宣言は同じ '{', '}'。特に区別はしていない。同じものを場合によって呼び方を変えている。

    foo : {
        ... ここにいろいろ書く
    } ;

    Woo : {
        ... コンストラクタは普通に記述、関数と同じ

        dtor : {
            ... デストラクタも用意してみた。名前は final にするか悩み中。
        }
    } ;

    ret : foo() ; ... 関数 foo の戻り値を ret に格納
    ins : Woo() ; ... class Woo のインスタンス ins を生成



// 名無しの class ＆関数もおっけー。

    {
        ... ここにいろいろ書く
    } ;

    {
        ... コンストラクタ

        dtor : {
            ... デストラクタ
        }
    } ;

// ちなみに、上書き可能な関数は↓。変数宣言の要領。

    foo :: {   } ; ... 最初の関数はこれ。上書き可能な変数。

    foo = {   } ; ... こっちの関数に置き換わる



// 引数はいわゆるメンバ変数として値が渡ってくるイメージ
// arg_size は引数の数（予約語です）

    foo : {
        @.co << _a ;       ... 第１引数
        @.co << arg_size ; ... 1
    } ;
    foo( a:1 ) ; ... 引数はいわゆるSTLの map,　必ず名前が必要。

    woo : {
        @.co << _x ; ... 第１引数
        @.co << _y ; ... 第２引数
        @.co << _z ; ... 第３引数
        @.co << arg_size ; ... 10
    } ;
    woo( x:1, y:2, z:3, a:4, b:5, c:6, d:7, e:8, f:9, g:10 ) ;



// 戻り値は return 文。複数の値が返せます。が、１つや無しでも list もしくは map が返ります。
// list の場合は .0 、map の場合は .xx を使って戻り値を参照する必要があります。

    return x:123, y:456 ; ... 基本（引数と同じく）map で返す

    return 123 ; ... 1つの場合

    return 123, 456 ; ... 複数の場合

    foo : {
        return a:10, b:20, c:30 ;
    } ;
    r : foo() ;
    @.co << r.b ; ... 20



    a == b ?? return ( c == b ?? 1 !! 2 ) ;

    a == b ?? return 1, 2, 3, ;

    a == b ?? return x:1, y:2, z:3, ;



// class 内メンバ変数の表記
// 基本 private、publicにしたい場合は変数名の先頭に '.' をつける。

    Foo : {
        x  :: 123 ;  ... private
        .y :: 456 ;  ... public
    } ;

    foo : Foo() ;
    foo.x = 0 ;  ... NG
    foo.y = 0 ;  ... OK



// class 内メソッドの表記も同様（メソッドとclassの区別は無いですが）

    Foo : {
        x : {
            ... private
        } ;
        .y : {
            ... public
        } ;
    } ;

    foo : Foo() ;
    foo.x() ;  ... NG
    foo.y() ;  ... OK

■コンテナ '[', ']' JavaScript ではおなじみの概念。

// STLの std::vector, std::map　が混在している

    int_list : [ 1, 2, 3, 4, 5, ] ;
    str_list : [ '1', '2', '3', ] ;
    str_map  : [ 1:'one', 2:'two', ] ;

    @.co << int_list[0] ; ... 1
    @.co << str_list[2] ; ... 2
    @.co << str_map[2] ;  ... two



// switch 文は無いので関数で実装。

    sw : [                      ... 固定テーブル（マップ）
        1 : { @.co << "one" },  ... func {} です。 () で書くと先に実行しちゃいます。
        2 : { @.co << "two" },
        unmatch : { @.co << it },  ... 一致しなかった時に呼ばれる関数（unmatchは予約語です）
    ] ;
    @.co << sw[2]() ; ... two ここで実行されます。


    sw :: [] ; ... 可変テーブル（マップ）、後から追加・変更可能。
    sw[1] = { @.co << 'one' } ;
    sw[2] = { @.co << 'two' } ;
    sw[unmatch] = { @.co << it } ;  ... 一致しなかった時に呼ばれる関数
    @.co << sw[2]() ; ... 'two'


    sw :: [] ; ... 可変テーブル（マップ）、後から追加・変更可能。
    sw << ( 1 : { @.co << 'one' } ) ;
    sw << ( 2 : { @.co << 'two' } ) ;
    sw << ( unmatch : { @.co << it } ) ;  ... 一致しなかった時に呼ばれる関数
    @.co << sw[2]() ; ... 'two'



// if 文でもいいんじゃないかな。

    i == 1 ?? @.co << 'one' !!
    i == 2 ?? @.co << 'two' !! @.co << i ;
