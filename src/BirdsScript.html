-----------------------------------------------------------------
■ 定数/変数宣言

    予約語、というか演算子は次の２つ。 ':', '::'

    必ず初期値が必用。空の変数や定数は存在させないことで、バグが入り込まないようにする設計方針。
    定数 いわゆる C/C++の const、Javaの final、Scalaの val 。 一度設定した値を上書きすることはできません。

    古いC言語や Pascal のように、スコープの先頭のみという制限はありません。好きなところで宣言できます。
    なるべく直前で定義して、素早く片づけることがよいコードになると思っています。



    ':' は定数の宣言、上書きするには変数宣言 '::'を使います。 いわゆる var。
    変数の方が制約が緩い（危険）なので、ハイコストにしています。上手に使ってください。


-----------------------------------------------------------------
■ if 文　条件分岐

    'if', 'else' という予約語はありません。
    C言語の三項演算子のような演算子 '??', '!!' を使います。

    最初は '?', '!' のひと文字だったのですが、文中に埋もれてしまって目立たないので二文字になりました。 



    '??' の前は必ず比較文である必要があります。比較文以外はsyntax-errorになります。
    flag = true ?? ... のようなタイポを防ぐ理由があります。

    true, false は予約語です。true, false の型は enum になる予定。



    現在のところ else文のみの書き方はNGになっています。
    メリットが感じられるようなら実装を検討します。そんなに難しい事ではないはず。




-----------------------------------------------------------------
■ スコープ（ブロックスコープ）

    いわゆるC言語やJavaの '{','}' ですが表記は '(', ')' になります。注意してください。

    変数/定数の参照範囲が制限できるのと、リソースの開放ができるので上手に使ってください。

    BirdsScript にはスコープのようにカッコで囲む構文が５種類あります。
    のちほど順番に解説しますが、まずは一覧と簡単な概要を紹介します。

    1, スコープ    (  )  実行の優先順位を示します。その場で実行され値を１つ返します。
    2, リスト      [  ]  いわゆる便利な配列です。その場で実行され list-objectを返します。
    3, ループ     {{  }}  範囲内を繰り返し実行します。その場で実行され list-objectを返します。
    4, each     .{{  }}  引数のある繰り返し文です。その場で実行され list-objectを返します。
    5, 関数宣言  {  }   関数を返します。呼ばれるまで実行はされません。実行後に list-objectを返します。



    スコープ '(', ')' に値を持たせることができます。最後の値がひとつだけ返ります。



    複数の値を返したい場合は list-objectを使ってください。
    list-objectは次項で説明します。



-----------------------------------------------------------------------
■ リスト '[', ']'。　list-object いわゆる便利な配列です。

    BirdsScript でサポートしている便利な配列は「リストオブジェクト(list-object)」と呼ぶことにしました。
    配列であり、連想配列でもあり、オブジェクトでもある汎用的な入れ物です。

    C++/STLの std::vector, std::map の両方の機能が混在しているコンテナになります。



    配列も、宣言の違いで、追加可能/不可能が変わります。



    ２次元配列も木構造も作れます。



    実は区切り記号 ',' を書かなくても通ります。が、何度も書きますが可読性（メンテナンス性）を優先してください。



-----------------------------------------------------------------------
■ switch文は list-object で書きます

    ちょっと脱線
    switch文は無いので関数の　list-objectで実装。



    個人的には switch文の代わりに if文の羅列でもいいんじゃないかと思う。



-----------------------------------------------------------------
■ ループ　'{{', '}}'　　繰り返し（その１）　for文

    予約語 break[], continue

    ループは'{{', '}}'であらわします。意味としては無限ループ for(;;){ ... } です。

    ちなみに無限ループと完全に等価ではなくて、リミット上限に到達すると例外が発生して勝手にループを抜けます。
    また break[] 文が無いとワーニングの予定（未実装）。できる限り安全側に倒したい（希望）。



    break[] は値を複数返すことができます。１つも、無しも、返せます。



    continue文のサンプルコード。



    ループを抜けると list-object を返します。実行した結果が配列の中に格納されまとめて返されます。




-----------------------------------------------------------------
■ each文 '.{{', '}}'　繰り返しその２、each 文

    繰り返し構文といえば each でしょ。 for文は break[] する必要があるのでできることなら使いたくない。

    each文は '.{{', '}}' で表します。
    'it', 'idx' は繰り返し時に暗黙的に定義され、値が渡される変数です。予約語になります。
    it : イテレータの意味。list-object の場合は各要素の値が入ります。
    idx : インデックスの意味。list-object の場合は各要素の名前が入ります。



    it も idx も定数扱いなので更新することはできません。



    eachではなく、ループ文にも idx があります。it は無いです。



    list-objectと each の組み合わせ



    for文と同じく、each文でも break[]は値を返すことができます。戻り値も同じく list-object。



    each文も繰り返し実行した結果を list-objectに格納して返します。



    list-objectに値を返す時に次のような注意が必要です。

    １回のループ処理で、複数の値を list-objectに積むことができる。
    もしくは、積まないこともできる。その「積む」「積まない」を決める構文が、 ',' と ';' です。

    ',' 値を list-object に積む。
    ';' 値をクリアするので list-object には積まない。



    また、 break[], continue の動作も list-object に積む/積まないが関係します。

■ range '..'   範囲を指定した list-object の表記

    range 表記 '..' は、 list-object の簡易表記です。

    1..2 は [ 1, 2, ] と等価です。
    3..6 は [ 3, 4, 5, 6, ] と等価です。
    9..5 は [ 9, 8, 7, 6, 5, ] と等価です。


    バグっている？↓
    @.co << 3.{{ [ it ] }}  ... [ [ 0, ], [ 1, ], [ 2, ], ]





    C言語のように文字を計算するのは、やはりちょっと抵抗がある

    c : 'String' ;
    d : c.{{
        out << it - ( it >= 'a' && it <= 'z' ?? ( 'a' - 'A' ) !! 0 ) ; ... out は変数なので上書き可能
    }} ;
    @.co << d ; ... d : 'STRING'

    ↓ 仕様の変更を検討中

    c : 'String'.to_int_list ;
    d : c.{{
        it - (
            it >= 'a' && it <= 'z'
            ?? 'a' - 'A'
            !! 0
        )
    }} ;
    @.co << d ;      ... [ 'S', 'T', 'R', 'I', 'N', 'G', ]
    @.co << d.join ; ... 'STRING'

■ 関数とオブジェクトの宣言は同じ '{', '}'。特に区別はしていない。同じものを場合によって呼び方を変えているだけ。

    予約語： dtor  デストラクタ



    名無しの関数やオブジェクトもおっけー。



    ちなみに、上書き可能な関数は↓。変数宣言の要領。



-----------------------------------------------------------------------
■ 関数呼び出しと引数

    関数呼び出しは定数宣言と同様に名前を付ける必要があります。

    f( 0, 1, 2 ) ; ... NG
    f( a:0, x:1, y:2 ) ; ... OK

    名前は、関数内の記述を確認してあわせてください。
    引数の数や型のチェックはあえてしていません（ジェネリックな書き方が出来るように）

    引数はいわゆるメンバ変数として値が渡ってくるイメージ
    f( a:0, x:1, y:2 ) と呼ぶと関数内に _a:0; _x:1; _y:2; が宣言されたのとほぼ同じ。

    予約語：  arg_size は引数の数

    memo: arg_size って、いりますかね。使わない気もする。
    memo: 名前なしでも _0, _1, ... で参照可能にするアイデアもありましたが、可読性＆メンテナンス性を考えていまのところ却下です。



-----------------------------------------------------------------------
■ return[] と戻り値

    戻り値は return[] 文。複数の値が返せます。いや、list-objectしか返せません！注意が必要です。

    うっかり .0 を忘れると list-objectそのものを参照してしまいます。

    予約語： return[]

■ オブジェクト内メンバ

    オブジェクト内メンバの表記
    基本 privateになります。
    publicにしたい（外から見せたい）場合は変数名の先頭に '.' をつける（記述コストで安全側に倒しています）



    オブジェクト内メソッドの表記も同様（メソッドとオブジェクトの区別は無いですが）



-----------------------------------------------------------------------
■ 型 .type

    ここでは型の説明をします。

    BirdsScript では、強い型付け（コンパイル時に型が決まり、代入や比較時に型のチェックをする）を目指しています。

    また、すべての変数や定数、そして関数や配列も型を持っています。

    それぞれの型を確認するには .type を使います。

    int　 整数
    str  文字列
    func　関数
    list　list-object
    enum 列挙型(boolも含まれる予定)
