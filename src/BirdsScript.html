以下、「BirdsScript」の文法です
-----------------------------------------------------------------
■コメント

    '---' いわゆる C++ の // と同じ
    '...' 上と同じ

-----------------------------------------------------------------
■ 定数/変数宣言

// 予約語は ':', '::'
// 必ず初期値が必用。空の変数や定数は存在しない（させない）ことで、バグが入り込まないようにする設計方針。
// 定数 いわゆるC++の const 。 Scalaの val 。 一度設定した値を上書きすることはできません。



// ':' は const なので定数の宣言、上書きするには変数宣言 '::'。 いわゆる var。
// 変数の方が制約が緩い（危険）なので、ハイコストにしています。上手に使ってください。


-----------------------------------------------------------------
■ if 文

// 'if', 'else' という予約語はありません。C言語の三項演算子のような '??', '!!' を使います。



// ?? の前は必ず比較文であること。比較文以外はエラーになります。
// flag = true ?? ... のようなタイポを防ぐ理由があります。



// 現在のところ else 文のみの書き方はNGになっています。メリットが感じられるようなら実装を検討します。




-----------------------------------------------------------------
■ スコープ（ブロックスコープ）

// いわゆるC言語の '{','}' ではなくて '(', ')' になります。
// 変数/定数宣言のスコープが制限できるので上手に使ってください。

// memo : ( ... ) は [ ... ].0 と意味的に同じにしたい。



// スコープ'(',')'に値を持たせることができます。
// 予約語は'out'。return文に似ているけど、処理は中断されないので注意。
// すみません。out はまだ使わないでください。仕様検討中です。

■ 繰り返し構文、for 文、each 文

// 予約語 break, continue

// '{{', '}}' はループ文 -> for(;;){ ... }。
// break は値を返すことができる。無しも可。
// ちなみに無限ループではなくて、リミット上限に到達すると例外が発生してループを終わります。
// break文が無いとワーニングの予定（未実装）。できる限り安全側に倒したい。



// '.{{', '}}' は each, 'it' はループ内で使用できる変数（イテレータの意味：  it は予約語です）
// 変数のイテレータであれば、上書き可能です。

    5.{{ @.co << it << ', ' ; }}  ... 0, 1, 2, 3, 4,

    5.{{ @.co << it << ', ' ; }}  ... 0, 1, 2, 3, 4,
    ↓ 仕様の変更を検討中
    @.co << 5.{{ it, }}  ... [ 0, 1, 2, 3, 4, ]

    @.co << 5..7.{{ it, }}  ... [ 5, 6, 7, ]



    c : 'String' ;
    d : c.{{
        it >= 'a' && it <= 'z'
        ?? out << it - ( 'a' - 'A' ) ; ... out は変数なので上書き可能
    }} ;
    @.co << d ; ... d : 'STRING'
    ↓ 仕様の変更を検討中
    c : 'String' ;
    d : c.{{
        it - (
            it >= 'a' && it <= 'z'
            ?? 'a' - 'A'
            !! 0
        ),
    }} ;
    @.co << d ; ... d : [ 'S', 'T', 'R', 'I', 'N', 'G', ]



// for ( int i=0 ; i<10 ; i++ ) ... をあえて書くなら



// 以下、例外と同等な機能なので・・・実装検討中

// 多重ループを抜ける break, continue (構造化された goto)

    {{
        label_a <- {} ;  ... label_a の関数宣言と同等。
        ...
        {{
            break label_a ; ... 関数 label_a をコールして,その関数のあるスコープの外へ飛ぶ
        }}

    }}  ... <- ここへ飛ぶ（外へ抜ける）

    {{
        bb <- {} ;
        ...
        {{
            continue bb ; ... 関数 bb をコールして, bb のあるスコープの最初に飛ぶ
        }}
    }}


// ループを抜ける break, continue (構造化された goto)

    (
        a <- {} ;
        ...
        {{
            break a ; ... 関数 a をコールして, a のあるスコープの外へ飛ぶ
        }}

    )  ... <- ここへ飛ぶ（外へ抜ける）

    (
        bb <- {} ;
        ...
        {{
            continue bb ; ... 関数 bb をコールして, bb のあるスコープの先頭に飛ぶ
        }}
    )

■ class と関数の宣言は同じ '{', '}'。特に区別はしていない。同じものを場合によって呼び方を変えているだけ。

// 予約語： dtor  デストラクタ



// 名無しの class ＆関数もおっけー。



// ちなみに、上書き可能な関数は↓。変数宣言の要領。

■ 関数呼び出しと引数
// 関数呼び出しは、定数宣言と同様に名前を付ける必要があります。
// f( 0, 1, 2 ) ; ... NG
// f( a:0, x:1, y:2 ) ; ... OK
// 名前は、関数内の記述を確認してあわせてください。
// 引数の数や型のチェックはあえてしていません（ジェネリックな書き方が出来るように）

// 引数はいわゆるメンバ変数として値が渡ってくるイメージ
// f( a:0, x:1, y:2 ) と呼ぶと関数内に _a:0; _x:1; _y:2; が宣言されたのとほぼ同じ。
// 予約語：  arg_size は引数の数

// memo : 名前なしでも _0, _1, ... で参照可能にするアイデアもありましたが、可読性＆メンテナンス性を考えていまのところ却下です。

■ class内メンバ

// class 内メンバの表記
// 基本 privateになります。
// publicにしたい（外から見せたい）場合は変数名の先頭に '.' をつける（記述コストで安全側に倒しています）



// class 内メソッドの表記も同様（メソッドとclassの区別は無いですが）

// 記述のお作法として class は先頭文字を大文字、 メンバや関数は先頭文字を小文字でお願いします。
// ラクダ記法も好きではないので、小文字の場合は _ を使ってください。

■配列 STLでいうコンテナ '[', ']' JavaScript ではおなじみの概念。

// STLの std::vector, std::map  が混在しているコンテナ



// 追加可能の有無も宣言の違いがあります。






//  ちょっと脱線
// switch 文は無いので関数の map で実装。

    sw : [                      ... 固定テーブル（マップ）
        1 : { @.co << "one" },  ... func {} です。 () で書くと先に実行しちゃいます。
        2 : { @.co << "two" },
        unmatch : { @.co << it },  ... 一致しなかった時に呼ばれる関数（unmatchは予約語です）
    ] ;
    @.co << sw[2]() ; ... two ここで実行されます。


    sw :: [] ; ... 可変テーブル（マップ）、後から追加・変更可能。
    sw[1] = { @.co << 'one' } ;
    sw[2] = { @.co << 'two' } ;
    sw[unmatch] = { @.co << it } ;  ... 一致しなかった時に呼ばれる関数
    @.co << sw[2]() ; ... 'two'


    sw :: [] ; ... 可変テーブル（マップ）、後から追加・変更可能。
    sw << ( 1 : { @.co << 'one' } ) ;
    sw << ( 2 : { @.co << 'two' } ) ;
    sw << ( unmatch : { @.co << it } ) ;  ... 一致しなかった時に呼ばれる関数
    @.co << sw[2]() ; ... 'two'


// if 文でもいいんじゃないかな。

    i == 1 ?? @.co << 'one' !!
    i == 2 ?? @.co << 'two' !! @.co << i ;
