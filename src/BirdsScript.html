■ 定数/変数宣言

// 予約語は ':', '::'

// 必ず初期値が必用。空の変数や定数は存在しない（させない）ことで、バグが入り込まないようにする設計方針。
// 定数 いわゆる C/C++の const、Javaの final、Scalaの val 。 一度設定した値を上書きすることはできません。

// 古いC言語や Pascal のように、スコープの先頭のみという制限はありません。好きなところで宣言できます。
// なるべく直前で定義して、素早く片づけることがよいコードになると思っています。



// ':' は const なので定数の宣言、上書きするには変数宣言 '::'。 いわゆる var。
// 変数の方が制約が緩い（危険）なので、ハイコストにしています。上手に使ってください。


-----------------------------------------------------------------
■ if 文

// 'if', 'else' という予約語はありません。C言語の三項演算子のような '??', '!!' を使います。



// ?? の前は必ず比較文であること。比較文以外はエラーになります。
// flag = true ?? ... のようなタイポを防ぐ理由があります。

// true, false は予約語です。true, false の型は enum になる予定。



// 現在のところ else 文のみの書き方はNGになっています。メリットが感じられるようなら実装を検討します。




-----------------------------------------------------------------
■ スコープ（ブロックスコープ）

// いわゆるC言語の '{','}' ですが、表示は '(', ')' になります。注意してください。

// 変数/定数宣言のスコープが制限できるので上手に使ってください。



// スコープ'(',')'には１つだけ値を持たせることができます。

// すみません。out ↓はまだ使わないでください。仕様を再検討中です。
// 予約語は'out'。return文に似ているけど処理は中断されないので注意。



-----------------------------------------------------------------------
■配列 '[', ']'。

// ここ BirdsScript では「リストオブジェクト(list-object)と呼ぶことにしました。 
// 配列でもあり、連想配列でもあり、オブジェクトでもある汎用的な入れ物なので。

// STLの std::vector, std::map の両方の機能が混在しているコンテナ。



// 配列も、宣言の違いで、追加可能/不可能が変わります。



// ２次元配列も、木構造も、作れます。



// ちょっと脱線
// switch 文は無いので関数の配列(list-object)で実装。



// switch の代わりに if 文の羅列でもいいんじゃないかと、個人的には思う。



-----------------------------------------------------------------
■ 繰り返し構文、for 文、each 文

// 予約語 break, continue

// '{{', '}}' はループ文 -> for(;;){ ... }。

// ちなみに無限ループではなくて、リミット上限に到達すると例外が発生してループを終わります。
// break文が無いとワーニングの予定（未実装）。できる限り安全側に倒したい。

// たとえば C言語風の for ( int i=0 ; i < 10 ; i++ ) ... をあえて書くなら



// continue 文のサンプルコード。 FizzBuzz 問題。



// break は値を返すことができる。無しも可。
// memo : return とおなじく 配列　[　]　のみを返す仕様のほうがいいかも



// 繰り返し構文といえば each でしょ。 for 文は break する必要があるのでできることなら使いたくない。

// each　は '.{{', '}}'。 'it' はループ内で使用できる変数（イテレータの意味。it は予約語です）
// 変数のイテレータであれば、上書き可能です。

    5.{{ @.co << it << ', ' }}  ... 0, 1, 2, 3, 4,

    5.{{ @.co << it << ', ' }}  ... 0, 1, 2, 3, 4,
    ↓ 仕様の変更を検討中
    @.co << 5.{{ it, }}  ... [ 0, 1, 2, 3, 4, ]

    @.co << 5..7.{{ it, }}  ... [ 5, 6, 7, ]



// 配列(list-object)と each の組み合わせ



// for 文と同じく、each文でも break は値を返すことができる。



// 以下実験中。仕様が変わる可能性が高いです。

// C言語のように文字を計算するのは、やはりちょっと抵抗がある

    c : 'String' ;
    d : c.{{
        out << it - ( it >= 'a' && it <= 'z' ?? ( 'a' - 'A' ) !! 0 ) ; ... out は変数なので上書き可能
    }} ;
    @.co << d ; ... d : 'STRING'

    ↓ 仕様の変更を検討中

    c : 'String'.to_arry ;
    d : c.{{
        it - (
            it >= 'a' && it <= 'z'
            ?? 'a' - 'A'
            !! 0
        ),
    }} ;
    @.co << d ; ... d : [ 'S', 'T', 'R', 'I', 'N', 'G', ]



// 以下、例外と同等な機能なので・・・実装検討中

// 多重ループを抜ける break, continue (構造化された goto)

    {{
        label_a -> {} ;  ... label_a の関数宣言と同等。
        ...
        {{
            break label_a ; ... 関数 label_a をコールして,その関数のあるスコープの外へ飛ぶ
        }}

    }}  ... <- ここへ飛ぶ（外へ抜ける）

    {{
        bb -> {} ;
        ...
        {{
            continue bb ; ... 関数 bb をコールして, bb のあるスコープの最初に飛ぶ
        }}
    }}


// ループを抜ける break, continue (構造化された goto)

    (
        a -> {} ;
        ...
        {{
            break a ; ... 関数 a をコールして, a のあるスコープの外へ飛ぶ
        }}

    )  ... <- ここへ飛ぶ（外へ抜ける）

    (
        bb -> {} ;
        ...
        {{
            continue bb ; ... 関数 bb をコールして, bb のあるスコープの先頭に飛ぶ
        }}
    )

■ 関数呼び出しと引数

// 関数呼び出しは、定数宣言と同様に名前を付ける必要があります。
// f( 0, 1, 2 ) ; ... NG
// f( a:0, x:1, y:2 ) ; ... OK
// 名前は、関数内の記述を確認してあわせてください。
// 引数の数や型のチェックはあえてしていません（ジェネリックな書き方が出来るように）

// 引数はいわゆるメンバ変数として値が渡ってくるイメージ
// f( a:0, x:1, y:2 ) と呼ぶと関数内に _a:0; _x:1; _y:2; が宣言されたのとほぼ同じ。
// 予約語：  arg_size は引数の数

// memo : 名前なしでも _0, _1, ... で参照可能にするアイデアもありましたが、可読性＆メンテナンス性を考えていまのところ却下です。

■ 型 .type

// ここでは型の説明をします。

BirdsScript では、強い型付け（コンパイル時に型が決まり、型のチェックをする）を目指しています。

また、すべての変数や定数、そして関数や配列も型を持っています。

それぞれの型を確認するには .type を使います。
