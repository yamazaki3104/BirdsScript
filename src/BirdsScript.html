つい、カッ！となってLL言語を作りはじめてしまいました。名前は「BirdsScript」です。
（2013/08/17 現在、未完成。）

特徴：

すべてのプログラマーのために「軽量で」「安全で」「大規模開発に耐えられる」「使って楽しい」言語を目指して設計してみました。
（C/C++ や JavaScript に近い文法だと思っています）


「軽量で」

　・記述量が軽量（ var とか function とか class とか private とか public とか書く必要はありません。型名も　int とか string とか （いまのところ）ほとんんど記述する必要がない。）

　・文法も軽量。予約語も少ない。（ if, for, each, switch, などは存在せず、代替の記述方法があります ）

　・実装も軽量。コンパイラ(Yamasemi)も、実行系(Kawasemi)も、JavaScriptで1000行程度。最終的にはC言語で5000行程度になるはず。

「安全で」

　・定数宣言を重視（不必要な変数はバグの元となると考えて）

　・空の変数は存在できない（変数の値が未定という状態は無い）

　・コンテナにも更新の可、不可がある。

　・メンバは普通 private になる（デフォルトでカプセル化、隠蔽され、外部からのアクセス可能にするには手間'.'の記述が必要）

　・ポインタは存在しない（ポインタはバグの温床という考え）

　・継承は存在しない（実装の継承は過度に隠ぺいされて可読性が悪いと考えて）

　・GCではなく、スコープで解放され、デストラクタが走ります（GCは処理が重いときに走り始めるので嫌いです）

　・例外を多数実装。無限ループする前に例外が発生してループを抜けます。（未実装ですが、ゼロ除算、最大最小値超え、範囲外アクセス、スタックメモリ枯渇、などなど）

　・タイプセーフな言語にしたい思いは強い（希望＆未実装）

「大規模開発に耐えられる」

　・オブジェクト指向（クラスとか、オブジェクトとか、カプセル化とか）の概念は入っている

　・STLのようなジェネリックな言語にしたい思いは強いです（希望＆未実装）。

　・テストも重視。デバッグも重視。（未実装）

　・可読性もメンテナンス性も重視。（希望）

「使って楽しい」

　・コンテナがある（配列も連想配列もごちゃ混ぜなコンテナ）

　・関数とクラスは同じ記述、同じ実装。

　・無名関数がある。無名クラスがある。

　・クロージャーも・・・？

とにかく、プログラマが苦労しない言語を目指しています。


概要：

コンパイラというか中間言語へのトランスレータ「Yamasemi」と、中間言語の実行部(いわゆるVM)「Kawasemi」の２段構成になっています。

現状、言語設計+試験期間な意味からコンパイラよりもインタープリタに近い状態です。現在は実装の確認という意味で JavaScript 書いています。
今後の予定としては、やはり C/C++ への移植です。JavaScriptで 1000行程度なので、C/C++ でも 4000行程度の軽量な実装になると予想しています。

型は現在、整数型(int)と文字列型(str)しかありません。早めに、bool, enum はサポートしたいと思っています。
float やその他の型は外部のライブラリとして用意するつもり。（言語そのもののの実装を軽くしたいので。）

クラスやオブジェクトのようなものは JavaScript と同じようなプロトタイプで用意する方向です。
継承（実装の継承は可読性とメンテナンス性が悪い）は嫌いなので、おそらく永遠にサポートされないでしょう。

逆に STL のようなジェネリックプログラミングは優先度高めでサポートします。
コンテナというか「リスト（配列）」や「マップ（連想配列）」は優先度を上げて実装しています。現状は中途半端に実装されています。

そしてテストとデバッグのしやすさを売りにしたいので test() や assert(), trace() とかを早めに実装したい。

以下、「BirdsScript」の文法です

■コメント

    '---' いわゆる C++ の // と同じ
    '...' 上と同じ

■ 定数/変数宣言

// 予約語は ':', '::'
// 必ず初期値が必用。空の変数や定数は存在しない（させない）ことで、バグが入り込まないようにする設計方針。
// 定数 いわゆるC++の const。Scalaの val とも言う。値を変えることができません。

    i : 0 ; ... 定数宣言 val

    str_n : "string\n" ; ... エスケープ有りの文字列

    str : 'string' ; ... エスケープ無しの文字列

// ':' は const なので定数の宣言、上書きするには変数宣言 '::'。 いわゆる var。
// 変数の方が制約が緩い（危険）なので、ハイコストにしています。上手に使ってください。

    i :: 0 ; ... 変数宣言 var

    str :: "string" ;

■ if 文

// 'if', 'else' という予約語はありません。C言語の三項演算子のような '??', '!!' を使います。

    foo < 0 ?? foo = 0 ;

    foo < 0 ?? foo = 0 !! foo = 1 ;

    boo : ( foo < 0 ?? 0 !! 1 ) ; ... スコープとの組み合わせ


// ?? の前は必ず比較文であること。比較文以外はエラーになります。
//　flag = true ?? ... のようなタイポを防ぐ理由もあります。

    is_ok ?? a=0 !! a=1 ; ... NG

    is_ok == true ?? a=0 !! a=1 ; ... OK



// 現在のところ else 文のみの書き方はNGになっています。メリットが感じられるようなら実装を検討します。

    foo >= 0 !! foo = 0 ; ... NG




-----------------------------------------------------------------
■ スコープ（ブロックスコープ）

// いわゆるC言語の '{','}' ではなくて '(', ')' になります。
// 変数/定数宣言のスコープが制限できるので上手に使ってください。

    foo < 0
    ?? ( foo = 0 ; boo = 0 ; woo = 0 ; )
    !! ( foo = 1 ; boo = 1 ; woo = 1 ; )

    foo < 0
    ?? (
        foo = 0 ;
        boo = 0 ;
        woo = 0 ;
    )
    !! (
        foo = 1 ;
        boo = 1 ;
        woo = 1 ;
    )

// memo : ( ... ) は [ ... ].0 と意味的に同じにしたい。


// スコープ'(',')'そのものに値を持たせることもできます。
// 予約語は'out'。return文に似ているけど、処理は中断されないので注意。
// すみません。out はまだ使わないでください。仕様検討中です。

    a : (
        foo = 1 ;
        out : boo = 2 ;  ... この式が値が()の値になる
        woo = 3 ;
    ) ;
    @.co << a ;          ... a : 2 になる。 @.co は C言語の stdout の意味で仮実装中。

    b : (
        foo = 1 ;
        boo = 2 ;
        woo = 3 ;
        boo            ... outを書かなくても最後の';'を書かなければオッケー。
    ) ;
    @.co << b ;        ... b : 2 になる。

■ 繰り返し構文、for 文、each 文

// 予約語 break, continue 

// '{{', '}}' はループ文 -> for(;;){ ... }。
// break は値を返すことができる。無しも可。
// ちなみに無限ループではなくて、リミット上限に到達すると例外が発生してループを終わります。
// break文が無いとワーニングの予定（未実装）。できる限り安全側に倒したい。

    a : {{

        break 10 ;    ... ループを抜ける。 out = 10 ; break ; と同じ。

        continue ;    ... ここは実行されないけど。ループの先頭に戻る。

    }} ;
    @.co << a ; ... a : 10 ;



// '.{{', '}}' は each, 'it' はループ内で使用できる変数（イテレータの意味：　it は予約語です）
// 変数のイテレータであれば、上書き可能です。

    5.{{ @.co << it << ', ' ; }}  ... 0, 1, 2, 3, 4

    @.co << 5.{{ out << it << ', ' ; }}  ... 0, 1, 2, 3, 4
    ↓ 仕様の変更を検討中
    @.co << 5.{{ it, }}  ... 0, 1, 2, 3, 4

    b : 10.{{
        it == 5 ?? break 555 ;
    }} ;
    @.co << b ; ... b = 555



    c : 'String' ;
    d : c.{{
        it >= 'a' && it <= 'z'
        ?? out << it - ( 'a' - 'A' ) ; ... out は変数なので上書き可能
    }} ;
    @.co << d ; ... d : 'STRING'
    ↓ 仕様の変更を検討中
    c : 'String' ;
    d : c.{{
        it - (
	        it >= 'a' && it <= 'z'
	        ?? 'a' - 'A'
	        !! 0
	    ),
    }} ;
    @.co << d ; ... d : [ 'S', 'T', 'R', 'I', 'N', 'G', ]



// for ( int i=0 ; i<10 ; i++ ) ... をあえて書くなら

    i :: 0 ;
    {{
        ...
        i >= 10 ?? break ;
        i ++ ;
    }}



// 以下、例外と同等な機能なので・・・実装検討中

// 多重ループを抜ける break, continue (構造化された goto)

    {{
    	label_a <- {} ;  ... label_a の関数宣言と同等。
        ...
        {{
            break label_a ; ... 関数 label_a をコールして,その関数のあるスコープの外へ飛ぶ
        }}

    }}  ... <- ここへ飛ぶ（外へ抜ける）

    {{
    	bb <- {} ;
        ...
        {{
            continue bb ; ... 関数 bb をコールして, bb のあるスコープの最初に飛ぶ
        }}
    }}


// ループを抜ける break, continue (構造化された goto)

    (
    	a <- {} ;
        ...
        {{
            break a ; ... 関数 a をコールして, a のあるスコープの外へ飛ぶ
        }}

    )  ... <- ここへ飛ぶ（外へ抜ける）

    (
    	bb <- {} ;
        ...
        {{
            continue bb ; ... 関数 bb をコールして, bb のあるスコープの先頭に飛ぶ
        }}
    )

■ class と関数の宣言は同じ '{', '}'。特に区別はしていない。同じものを場合によって呼び方を変えているだけ。

// 予約語： dtor　デストラクタ

    foo : {
        ... ここにいろいろ書く
    } ;

    Woo : {
        ... コンストラクタは普通に記述、関数と同じ

        dtor : {
            ... デストラクタも用意してみた。名前は final にするか悩み中。
        }
    } ;

    ret : foo() ; ... 関数 foo の戻り値を ret に格納
    ins : Woo() ; ... class Woo のインスタンス ins を生成



// 名無しの class ＆関数もおっけー。

    {
        ... ここにいろいろ書く
    } ;

    {
        ... コンストラクタ

        dtor : {
            ... デストラクタ
        }
    } ;

// ちなみに、上書き可能な関数は↓。変数宣言の要領。

    foo :: {   } ; ... 最初の関数はこれ。上書き可能な変数。

    foo = {   } ; ... こっちの関数に置き換わる

■ 関数呼び出しと引数
// 関数呼び出しは、定数宣言と同様に名前を付ける必要があります。
// f( 0, 1, 2 ) ; ... NG
// f( a:0, x:1, y:2 ) ; ... OK
// 名前は、関数内の記述を確認してあわせてください。
// 引数の数や型のチェックはあえてしていません（ジェネリックな書き方が出来るように）

// 引数はいわゆるメンバ変数として値が渡ってくるイメージ
// f( a:0, x:1, y:2 ) と呼ぶと関数内に _a:0; _x:1; _y:2; が宣言されたのとほぼ同じ。
// 予約語：　arg_size は引数の数

    foo : {
        @.co << _a ;       ... 第１引数
        @.co << arg_size ; ... 1
    } ;
    foo( a:1 ) ; ... 引数はいわゆるSTLの map,　必ず名前が必要。

    woo : {
        @.co << _x ; ... 第１引数
        @.co << _y ; ... 第２引数
        @.co << _z ; ... 第３引数
        @.co << arg_size ; ... 10
    } ;
    woo( x:1, y:2, z:3, a:4, b:5, c:6, d:7, e:8, f:9, g:10 ) ;

// memo : 名前なしでも _0, _1, ... で参照可能にするアイデアもありましたが、可読性＆メンテナンス性を考えていまのところ却下です。

■ return と戻り値

// 戻り値は return 文。複数の値が返せます。いや、複数の値 list もしくは map しか返せません。注意が必要です。

// 特に、戻り値の参照のしかたに注意が必要です。
// list の場合は .0 、map の場合は .xx を使って戻り値を参照する必要があります。
// うっかり .0 を忘れると list や　map そのものを参照してしまいます。

// memo : であるならもう return の構文は return [ .. ] のみにしてしまってよい気がする。

// 予約語： return

    return x:123, y:456 ; ... 基本（引数と同じく） map として名前付きで返す （推奨）

    return 123 ; ... 1つの場合でも list として返る

    return 123, 456 ; ... 複数の場合

    return ; ... 無しの場合も 空の list として返る

    foo : {
        return a:10, b:20, c:30 ;
    } ;
    r : foo() ;
    @.co << r.b ; ... 20



    a == b ?? return ( c == b ?? 1 !! 2 ) ;

    a == b ?? return 1, 2, 3, ;

    a == b ?? return x:1, y:2, z:3, ;

// 追加可能の有無も宣言の違いがあります。

    list_a :  [] ; ... 追加不可なコンテナ
    list_b :: [] ; ... 追加可能なコンテナ

    list_a << 1 ;  ...  NG
    list_b << 1 ;  ...  OK

//　ちょっと脱線
// switch 文は無いので関数の map で実装。

    sw : [                      ... 固定テーブル（マップ）
        1 : { @.co << "one" },  ... func {} です。 () で書くと先に実行しちゃいます。
        2 : { @.co << "two" },
        unmatch : { @.co << it },  ... 一致しなかった時に呼ばれる関数（unmatchは予約語です）
    ] ;
    @.co << sw[2]() ; ... two ここで実行されます。


    sw :: [] ; ... 可変テーブル（マップ）、後から追加・変更可能。
    sw[1] = { @.co << 'one' } ;
    sw[2] = { @.co << 'two' } ;
    sw[unmatch] = { @.co << it } ;  ... 一致しなかった時に呼ばれる関数
    @.co << sw[2]() ; ... 'two'


    sw :: [] ; ... 可変テーブル（マップ）、後から追加・変更可能。
    sw << ( 1 : { @.co << 'one' } ) ;
    sw << ( 2 : { @.co << 'two' } ) ;
    sw << ( unmatch : { @.co << it } ) ;  ... 一致しなかった時に呼ばれる関数
    @.co << sw[2]() ; ... 'two'

    i == 2 ?? @.co << 'two' !! @.co << i ;
